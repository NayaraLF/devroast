# Formato de Specs

Specs devem ser criados em `specs/<feature-name>.md` antes de implementar.

## Estrutura

```
# Especificação: <Nome da Feature>

## 1. Visão Geral / Objetivo
Breve descrição do que será implementado.

## 2. Funcionalidades do Sistema
Lista de funcionalidades principais.

## 3. Especificação Técnica
- Estrutura de dados (tabelas, tipos)
- Componentes (props, interfaces)
- APIs/Serviços

## 4. TO-DO
- [ ] Tarefas a executar
- [x] Tarefas concluídas

## 5. Perguntas em Aberto
Questões a resolver antes/durante implementação.
```

## Regras

- Seja conciso, mas completo o suficiente para implementação
- Inclua código/exemplos quando necessário
- Documente decisões tomadas com ✅
- Mantenha TO-DO atualizado durante implementação
